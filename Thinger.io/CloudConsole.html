雲端控制台(CLOUD CONSOLE)

此節將介紹雲端控制台可以提供您哪些功能用於產生控制平台上的不同元素，如設備、端點、數據儲存桶或是存取令牌。

登入Thinger.io帳戶後，預設情況下將直接存取本頁面的統計(statistics)元素， 其將供您查看帳戶中的一些基本資料，如設備、端點、數據儲存桶以及儀表板的數量與上限資訊，並提供一張地圖用於顯示目前連接設備的大概位置。 位於螢幕下方，亦提供了設備所消耗之傳輸數據量的統計資料與圖表。

在螢幕左側，您將會看到主選單，其中包含所有此平台提供在建構物聯網項目時會用到的功能。以下各節將對各項功能詳加描述。

新增設備

設備設定

通過單擊設備儀表板右上角的齒輪圖示，可以調整某些設備詳細資訊，如描述或憑證。這樣，您可以通過您選擇的新設備更改設備憑證，以防忘記憑證（憑證無法從資料庫中恢復，因為它已加密）。請注意，更改設備憑證不會斷開設備，但會在斷開連接後阻止其重新連接。

設備API

Thinger.io平台的一個很酷的功能是，它可以探索設備中定義的資源。資源可以是傳感器讀數，如溫度，濕度或壓力。也可以是任何可操作的元素，如燈，繼電器或電機。一般來說，任何設備資源都像一個可以通過Rest API依照需求調用的回調函數。

通過這種方式，本節將介紹如何通過雲控制台與設備資源進行互動，同時您亦將了解如何調用自己的REST API來存取資源。

將設備連接到帳戶後，如上一節所述，您可以存取其資源並使用 API EXPLORER 探索API Rest端點。您可以通過單擊名為Device API的藍色小按鈕，在DEVICE標籤頁中存取此畫面。

在API資源管理器界面中，您將看到許多不同的框列出程式中定義的每個資源。每個資源都有一個ID，該ID與程式中定義的資源名稱相關。

在Thinger.io平台中，您可以定義4種不同類型的資源，分別為：輸入（向設備發送數據），輸出（設備將發送資訊），輸入/輸出（您可以在一個調用中發送和接收資訊）以及回調資源(僅用於執行函數而不傳遞任何數據)。從API的角度來看，輸入和輸出數據可以是任何JSON文件。您可以檢視函式庫文件了解如何定義這些不同的資源。

例如，Arduino程式庫中的預設ESP8266範例定義了兩種不同的資源。一個稱為led的輸入資源，用於控制BUILTIN_LED和一個調用millis以提取設備直行至目前毫秒數的輸出資源，如以下程式中所定義。請注意，資源名稱可以是用於標識基礎資源的任意文字，它們與平台中定義的任何常數無關。

以下這些是我們的範例資源：


	thing["led"] << digitalPin(BUILTIN_LED);
	thing["millis"] >> outputValue(millis());

如果我們的設備連接到平台，我們可以打開我們的設備API資源管理器並檢視平台中定義的資源，如下圖所示。

現在您可以看到如何於平台中使用設備中所定義的資源，因為設備能夠回報可用資源及其格式（或目前狀態）。您可以在這裡測試您的資源，即時與它們進行互動。在這種情況下，您將能夠切換LED狀態，或從Arduino設備讀取目前的毫秒數。

每次點擊Run按鈕都將立即執行您的資源，即根據資源類型，向設備要求數值、調用millis()函數或為執行機構發送新狀態，具體行為取決於資源類型（輸入或輸出）。

關於這一點的好處是每個設備資源都可以轉換為REST API端點，因此您可以使用標準REST請求如POST方法將值發送到設備，或使用GET從設備讀取資訊等方法來使用或與設備互動。因此，在這些API端點被集成到其他平台或應用程式之前即可輕鬆對這些端點進行測試。

您甚至可以從平台測試更複雜的輸入和輸出。在以下範例中，假設您已經定義了一個返回兩個整數和與乘積的資源，例如：


thing["in_out"] = [](pson& in, pson& out){
    out["sum"] = (int)in["value1"] + (int)in["value2"];
    out["mult"] = (int)in["value1"] * (int)in["value2"];
};

此資源定義將轉換為平台中的以下資源，其既可以測試輸入值，也可以檢視輸出結果。因此，您可以嘗試輸入一些值，單擊Run，然後檢視設備回報的輸出。此範例還強調資源如何工作，因為它們不僅僅是靜態值，而是可以使用任何輸入或輸出值調用的回調。

除了可以與設備互動的有用設備API資源管理器之外，您還可以通過單擊Show Query按鈕取得有關REST API端點的特定資訊。它提供方法(method)、URL、內容類型(ContentType)、請求內容(Request Body)和響應內容(Response Body)等資訊。您也可以點擊Curl，以便您可以直接複製指令與設備進行互動。

設備令牌(Device Token)

所有與您連接的設備的互動，即使用上面評論的REST API端點或智慧手機，都需要針對平台進行身份驗證。預設情況下，當您通過Thinger.io控制台與設備進行互動時，您將使用從用戶名和密碼獲取的存取令牌隱式地將所有請求簽核到平台。這種授權允許您存取所有的帳戶資源，因此您可以設定設備，數據儲存桶等。但此授權經常過期（將由瀏覽器自動續約），並且不能用於其他我們授予設備存取權限的用戶或平台，因為您將提供對所有帳戶的存取權限。

在這種情況下，可以建立特定的存取令牌以授予對設備的存取權限，甚至可以授予對設備上特定資源的存取權限。此外，通過設定有效日期，可以即時定義令牌有效性。這樣，如果您需要為第三方工具（如IFTTT，外部網頁，手機或其他任何服務）提供對某些設備資源的存取權限，強烈建議您建立設備令牌。

要建立設備令牌，請打開設備儀表板，然後檢視名為 "Device Tokens"的子頁面，單擊右側綠色的Add按鈕。然後，會出現一個視窗，您可以設定不同的參數：

  • Token name：使用代表名稱來記住令牌的發布原因，即IFTTT Access，手機等。
  • Token access：設定令牌以允許存取或限制對設備資源的存取。
  • Token expiration：將令牌設定為在某個給定日期到期，或者永不過期。

設備屬性

數據儲存桶(Data Buckets)

數據存儲桶是一種虛擬存儲，您可以在其中保留時間序列信息，例如隨時間變化的溫度或濕度。也可以使用它們存儲其他事件，例如移動偵測、車庫門打開、溫度警告等。

此數據可用於在儀表板中繪製資訊，或者以不同的格式導出進行外部處理。

建立儲存桶

要建立數據儲存桶，您需要單擊Data Buckets來使用該功能：

要建立數據儲存桶，只需按下Add Bucket按鈕，它將顯示以下畫面：

還有設定必要的參數：

  • Bucket Id：您的數據儲存桶的ID。
  • Bucket name：使用代表名稱來記住數據儲存桶，例如WeatherData。
  • Bucket description：在這裡填寫更多細節描述，如室內溫度和濕度。
  • Enabled：可以啟用或禁用數據儲存桶寫入。將其打開即可啟用寫入。
  • Data source：

    • From Device Resource：這意味著它將從特定設備資源（如溫度，運動等）獲取資訊。在此選項中，設備應與伺服器保持連接。這個功能所提供的好處是：我們可以根據需要更改Sampling Interval選項設定採樣率，而無需更動設備程式。

      請記住，此處更詳細地描述了如何在設備中定義資源，不過由DHT傳感器回報溫度和濕度的資源可以這樣寫： 

      
// define the resource just once in the setup
thing["TempHum"] >> [](pson &out){ 
	out["temperature"] = dht.readTemperature();
	out["humidity"] = dht.readHumidity();
};

      也可以讓設備特定條件時發起事件來串流傳輸資訊。在這種情況下，我們可以Update by Device在設定數據儲存桶時使用該選項，並使用此處所述的傳輸串流資源。

      使用前面的TempHum範例資源，如下面的程式片段那樣。

      
void loop() {
	thing.handle();
	// use your own logic here to determine when to stream/record the resource.
	if(requires_recording){
		thing.stream("TempHum");
	}
}

      這樣，數據儲存桶就訂閱了設備資源，並且在每次串流調用中都登錄了它的資訊。

    • From Write Call：此選項將設定數據儲存桶為在預設情況不會記錄任何資訊的狀態。它會等待調用寫入函數，如從Arduino的函式庫調用write_bucket方法或與Sigfox一樣直接調用REST API。此功能允許從不同設備寫入資料到同一數據儲存桶中，或者紀錄來自未與伺服器永久連接設備的資訊(處於睡眠模式的設備或使用Sigfox等其他技術的資訊)。

      以下是ESP8266設備使用write_bucket功能將資訊寫入數據儲存桶的範例：

      
void setup() {
  // define the resource with temperature and humidity
  thing["TempHum"] >> [](pson &out){ 
    out["temperature"] = dht.readTemperature();
    out["humidity"] = dht.readHumidity();
  };
}

void loop() { 
  // handle connection
  thing.handle();
  // write to bucket BucketId the TempHum resource
  thing.write_bucket("BucketId", "TempHum");
  // sleep the device SLEEP_MS milliseconds
  ESP.deepSleep(SLEEP_MS*1000, WAKE_RF_DEFAULT); 
}

檢視儲存桶數據

當設定了數據儲存桶並且開始記錄來自設備或調用寫入的數據，它即在表中顯示資訊。每條記錄都包含伺服器中的UTC時間戳（但以雲端平台中設定的時區時間顯示）和記錄值。存儲在數據數據儲存桶中的值可以是單個值，也可以是任何其他JSON文檔。如果是JSON文檔，則由鍵值對組成，如前面的範例所示，它們將以表格格式顯示，就像在下面的畫面截圖中一樣。

清除儲存桶數據

這個功能對於清除數據儲存桶資訊，而非刪除整個數據儲存桶並再次建立和設定它可能很有用。因此，您可以從數據儲存桶頁面輕鬆清除數據儲存桶或其中的一部分。在清除過程中，數據儲存桶仍然可以記錄設備中的資訊。

導出儲存桶數據

可以不同的文件格式導出所有儲存的資訊，因此您可以離線處理數據，例如應用人工智慧、業務分析、大數據等。您可以存取數據儲存桶並設定導出細節，比如選擇文件類型或導出範圍。幾分鐘後，您將收到一封附檔包含您所需資料的電子郵件（預設情況下，雲端平台中的資料有效期為3個月）。

儀表板(Dashboards)

儀表板是一個圖形用戶界面，允許以不同的圖形和圖表顯示您的資訊。您可以使用不同的小工具設定儀表板，設定其佈局，維度，顏色和數據源，以便為您的業務或流程產生有價值的資訊。

儀表板可以從您的設備即時顯示資訊（使用伺服器上的網頁框架以最小延遲），或使用存儲在定期輪詢的數據存儲區中的歷史資訊。可以單獨為每個儀表板視窗小工具設定數據源。對於連接到平台的設備，甚至可以動態地設定每個資源的採樣間隔，即在從傳感器讀數定義的資源中，它將允許調整其物理採樣間隔和通過線路的傳輸。儀表板不僅僅用於顯示數據，還可以通過連接的設備即時啟動，因此您可以使用一些控制小工具，如開/關值或滑塊。

在Thinger.io中，不僅可以在儀表板中顯示資訊，還可以即時控制設備。在本節中描述了一些可用於控制連接設備的可用小工具。

預設情況下，任何儀表闆對帳戶所有者都是私有的。但是，可以共享儀表板，以便其他人可以存取您的資訊。要共享儀表板，只需輸入儀表板設定並啟用Share開關即可。啟用儀表板共享後，將生成一個可以公開共享的URL。

注意：產生的授權（附加到URL末尾）可用於部分存取您的資源，例如用於提供圖表的設備或數據存儲區。檢視生成的存取令牌以獲取更多詳細資訊
注意：如果通過新增新數據源（設備或數據存儲區）來更改儀表板，則必須禁用重新啟用儀表板共享以更新授權。出於安全原因，它不會自動共享新數據源。

端點(Endpoints)

端點是服務，程序或其他目標的進入點。因此在Thinger.io中，端點可以定義為目標被設備調用以執行任何操作，例如發送電子郵件，發送SMS，調用REST API，與IFTTT互動，從不同的帳戶調用設備，或調用其他任何HTTP端點。

在小型微控制器中，通過設備直接調用這些端點可能很複雜，並且在設備中需要更多頻寬。通過這種方式，Thinger.io可以處理設備可以直接請求的端點呼叫，通過使用ID啟動它們並傳遞所需的任何資訊。它還增加了一些彈性，因為端點請求可以根據需要進行動態更改，而不更動設備中部署的程式。

電子郵件端點允許從您的設備發送電子郵件。您可以定義目標電子郵件地址，標題和撰寫電子郵件正文。

在下面的畫面截圖中，有一個電子郵件端點範例，其中包含一些文字和變數，這些文字和變數在設備調用端點時填充，並新增設備報告的目前溫度和濕度。請注意，temperature和humidity變數在雙括號內{{}}，所以終端將期望這些資訊完成主體。在下文中，有一些程式範例調用此端點。

基本上需要使用call_endpoint方法調用端點，其需要端點ID，在此範例中為ExampleEmail ，並且要發送可選數據到端點，這是一個pson文檔（非常類似於JSON）有兩個鍵(key)分別命名為temperature並humidity分別儲存DHT傳感器的讀數。以下將示範調用端點。


pson data;
data["temperature"] = dht.readTemperature();
data["humidity"] = dht.readHumidity();
thing.call_endpoint("ExampleEmail", data);
注意：如果要在電子郵件正文中包含單個值，則可以使用{{}}不帶任何鍵的雙括號，並pson使用單個值從設備發送文檔。如以下結構：

Temperature is: {{}} ºC

可以在設備中以此程式調用：


pson data = dht.readTemperature();
thing.call_endpoint("ExampleEmail", data);

HTTP端點

HTTP端點是一種通用類型的端點，可用於與任何其他Web服務或Web應用程式進行互動。因此，可以為此端點發出的任何HTTP請求設定方法(method)，URL，標頭和正文。

可設定的參數如下：

  • Request URL：設定方法（GET，POST，PUT，PATCH或DELETE）和欲請求的URL。
  • Request Headers：可以向請求新增標頭，這對於新增授權、暫存控制、設定內容類型等非常有用。
  • Request Body：正文可以是具有特定內容的自定義正文，也可以是具有設備發送的資訊的JSON有效負載(payload)。在自定義正文中，可以新增自定義變數，如電子郵件示例中所示。這樣，就可以建立不同格式的內容，如XML，SOAP等（請記得在這種情況下新增適當的內容類型(content-type）。

存取令牌(Access Tokens)

可以通過REST API調用存取所有Thinger.io後端。事實上，控制台只是一個與API交互的Angular REST客戶端，用於管理設備，數據儲存桶，端點，儀表板等。必須對每個REST API請求進行身份驗證才能生效，因此，任何客戶端都需要在每次調用時提供授權程式。這樣，存取令牌是向第三方服務或應用程式提供授權以進行API請求的方式，而無需共享用戶名和密碼。此外，通過存取令牌，可以授予對帳戶的特定資源和操作的存取權限，例如讀取特定設備或寫入自定義數據儲存桶（如本示例中所示）。
